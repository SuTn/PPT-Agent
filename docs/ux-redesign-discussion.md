## 一、产品定位

### 与 LandPPT 的差异化

| | LandPPT | PPT-Agent |
|---|---|---|
| 交互模式 | 面板驱动，填表式四阶段工作流（需求确认→大纲→任务板→生成） | 对话驱动，聊天式 |
| 用户心智 | "我在操作一个工具" | "我在和一个助手对话" |
| 学习成本 | 需要理解面板结构 | 自然语言，低门槛 |
| 编辑体验 | 专门的任务板和编辑面板 | 当前较弱，靠聊天指令 |

**结论：不走 LandPPT 的面板工作流路线。PPT-Agent 的差异化在于对话驱动。**

## 三、当前问题分析

### 问题 1：研究笔记——用户根本不会点开

**现状：** `ResearchPreview.vue`
- 默认折叠状态，只显示黄色图标 + "研究笔记" + "展开"按钮
- 点击后是一大段 markdown 渲染的文字（`max-height: 400px` 滚动块）
- 没有摘要、没有关键发现预览
- **用户没有动力点开，即使点开也看不下去**

### 问题 2：大纲——信息太多但没有层次感

**现状：** `OutlinePreview.vue`
- 所有页面平铺在一个列表里，12 页就是 12 行密集文字
- 数据模型本身不错（SCQA 框架、layout badge、evidence 标签）
- 但视觉呈现上：
  - `slide-item` 每个只有 `padding: 4px 8px`，非常紧凑
  - 标题、正文、论据、证据全挤在一起
  - 没有视觉重点，扫一眼就是一大块灰色文字区域

### 问题 3：工具调用卡片——只有状态没有内容

**现状：** `MessageList.vue` 中的 `tool-card`
- 研究过程中只显示 "正在研究主题..."
- 完成后只显示 "研究完成"
- 用户等待期间不知道 AI 在做什么
- 完成后不知道结果是什么

### 用户视角的完整体验

```
用户: "帮我做一份产品发布会PPT"
    ↓
[齿轮转] 正在研究主题...          ← 只有文字，没有内容预览
    ↓
[✓] 研究完成 → [研究笔记 卡片]    ← 默认折叠！要点开才看到一堆文字
    ↓
[齿轮转] 正在生成大纲...
    ↓
[✓] 大纲已生成 → [大纲卡片]       ← 12行密密麻麻的信息列表
    ↓
[齿轮转] 正在生成幻灯片...        ← 只有状态文字
    ↓
[缩略图条出现]
```

## 四、改进方案

### 4.1 研究阶段：从"折叠文字块"变成"发现流"

**目标：** 让用户看到 AI 正在发现什么，而不是事后展示一篇长文

**方案：**
- 研究过程中逐步显示关键发现卡片（3-5 条）
- 每条卡片：一个图标 + 一句话总结
  - 例："行业趋势：2025年AI产品发布数量增长40%"
  - 例："目标受众：25-35岁科技从业者，偏好数据驱动的呈现方式"
- 完成后显示简要摘要，全文可折叠查看

**视觉效果示意：**
```
┌──────────────────────────────────┐
│ 🔍 正在研究：产品发布会          │
├──────────────────────────────────┤
│ 📊 关键发现 1                    │
│ 行业趋势：AI产品发布数量增长40%   │
├──────────────────────────────────┤
│ 👥 关键发现 2                    │
│ 受众画像：25-35岁科技从业者       │
├──────────────────────────────────┤
│ 💡 关键发现 3                    │
│ 最佳实践：故事化结构效果提升60%   │
└──────────────────────────────────┘
```

### 4.2 大纲阶段：从"平铺列表"变成"可视化结构"

**目标：** 让用户一眼看到 PPT 的骨架，快速理解整体结构

**方案 A — 卡片网格：**
- 每张卡片代表一页幻灯片
- 卡片显示：页码 + 标题 + layout 标签
- 不同 layout 类型用不同视觉样式区分（封面/章节/内容/结尾）
- 点击某张卡片展开查看详情

**方案 B — 目录树 + 详情：**
- 左侧：缩略目录树，章节可折叠
- 右侧：选中某页时展示完整详情
- 类似文件管理器的双栏布局

**建议先采用方案 A（卡片网格），实现更简单，效果直观。**

**视觉效果示意：**
```
┌────────┐ ┌────────┐ ┌────────┐
│  封面   │ │  目录   │ │ 章节1  │
│ 产品发布│ │ 8页概览 │ │ 市场分析│
└────────┘ └────────┘ └────────┘
┌────────┐ ┌────────┐ ┌────────┐
│ 内容   │ │ 内容   │ │  内容   │
│ 竞品对比│ │ 用户画像│ │ 核心功能│
└────────┘ └────────┘ └────────┘
```

### 4.3 生成阶段：进度可视化

**目标：** 让等待过程不那么焦虑

**方案：**
- 幻灯片生成时，逐步显示每一页的缩略图在填充进来
- 不再只显示 "正在生成幻灯片..."
- 显示进度："正在生成第 3/12 页..."
- 每页生成后缩略图从灰占位变为实际预览

## 六、用户行为分析

### 两类典型用户
